# NetCtl v1.0.0 Release

**Release Date:** April 8, 2026  
**Status:** 🚀 Automated Build in Progress (GitHub Actions)

## Overview

NetCtl is a production-grade network management daemon with advanced capabilities:

- **eBPF/XDP QoS:** Kernel-level packet filtering and rate limiting
- **Transactional State:** Full rollback support for all network operations  
- **Web Dashboard:** Real-time metrics, device management, VLAN control
- **Interactive Setup:** TUI wizard for quick deployment
- **VLAN Management:** 802.1Q virtual network segmentation
- **DHCP Integration:** Automatic scope generation with dnsmasq

## Quick Start

### 1. Run Interactive Setup

```bash
sudo netctl-tui
```

The setup wizard guides you through:
- Network interface selection
- IP address configuration
- DNS settings
- Dashboard hostname
- Security review

### 2. Start the Daemon

```bash
sudo systemctl start netctl
sudo systemctl status netctl
```

Enable auto-start on boot:
```bash
sudo systemctl enable netctl
```

### 3. Access Dashboard

Open a browser and navigate to:
```
https://hostname:443
```

**Default Dashboard URL:**
- **Hostname:** netctl.local (configurable)
- **Port:** 443 (HTTPS, configurable)
- **Protocol:** HTTPS with self-signed certificate

### 4. Check Logs

```bash
# Real-time logs
sudo journalctl -u netctl -f

# Last 50 lines
sudo journalctl -u netctl -n 50
```

## Key Features

### Dashboard Tabs

**Overview**
- Connected device count

- Packet drop statistics
- Active VLAN count

**Devices**
- Discover and list connected devices
- Assign VLANs
- Apply rate limits
- Enable/disable devices

**VLANs**
- Create new virtual networks
- Configure DHCP scopes
- Delete VLANs
- View subnet information

**QoS**
- Apply bandwidth limits
- Block specific devices
- Set rate limiting rules
- Real-time rule management

**LAN Dashboard Configuration**
- Set hostname for remote access
- Configure port (default 443)
- Enable/disable HTTPS
- DNS verification
- Loop detection

### API Endpoints

NetCtl provides REST API for programmatic access:

```bash
# List devices
curl http://localhost:3001/api/devices

# List VLANs
curl http://localhost:3001/api/vlans

# Create QoS rule
curl -X POST http://localhost:3001/api/qos \
  -H "Content-Type: application/json" \
  -d '{"device_id": "...", "rate_limit": "10Mbps"}'
```

## Build Information

### Build Details

**Backend (Rust):**
- Async runtime: Tokio
- Database: SQLite with transactions
- Kernel interface: eBPF/XDP
- Protocol: REST API + SSE streaming

**Frontend (TypeScript/React):**
- Framework: React 18+
- Build: Vite
- UI: Matrix cyberpunk theme
- Real-time updates: Server-Sent Events

**eBPF Programs (C):**
- Kernel-level packet filtering
- XDP for NIC attachment
- CPU-efficient processing
- Zero-copy data plane

## Troubleshooting

### Package Won't Install

**Debian/Ubuntu:**
```bash
# Check dependencies
sudo apt update
sudo apt install -f

# Install manually with dependencies
sudo apt install -y dnsmasq libssl3
sudo apt install ./netctl_1.0.0-1_amd64.deb
```

**CentOS/RHEL:**
```bash
# Check dependencies
sudo yum check-update
sudo yum install -y dnsmasq openssl

# Install manually
sudo rpm -i ./netctl-1.0.0-1.el8.x86_64.rpm
```

### Service Won't Start

```bash
# Check service status
sudo systemctl status netctl

# View detailed logs
sudo journalctl -u netctl -n 100

# Manual start with debug output
sudo RUST_LOG=debug netctl
```

### Dashboard Access Issues

```bash
# Check network interface
ip link show

# Verify service is running
sudo netctl-tui  # Re-run setup wizard

# Check firewall
sudo ufw status
sudo firewall-cmd --list-all
```
